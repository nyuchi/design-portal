## Guiding philosophy

The Mukoko design system is built for **the reality of African infrastructure**. A user on a 2G connection in a village is not an edge case — that is the primary use case. Every foundation decision is shaped by four architectural pillars:

1. **Local-First** — data and UI must work offline, syncing when connectivity returns
2. **Mobile-First** — most users access Mukoko apps on mobile devices with constrained bandwidth and lower-end hardware
3. **Open Source & Sovereign** — technology sovereignty for Africa; no dependency on infrastructure we do not control
4. **Open Data** — transparent data pipelines that serve the continent's interests

These pillars translate into concrete constraints:

- **Language-inclusive** — the font stack supports African languages with full diacritics and extended character sets; African languages are first-class citizens, not localisations
- **Accessible by default** — every component meets APCA 3.0 AAA contrast standards with 48px minimum touch targets
- **Performant on constrained devices** — lazy loading, memory pressure monitoring, and sequential mount queues keep the experience smooth on 2G connections and low-RAM devices
- **Offline-capable** — components and patterns are designed for offline-first operation with graceful degradation

These are not aspirational goals. They are enforced constraints that shape every component in the registry.
