# Component Backlinks

Every component in the design system has a canonical URL on design.nyuchi.com that identifies it. The portal page and the JSON API are live today; per-component playground, health, chaos, and fundi endpoints are tracked for future work and surfaced honestly below.

## URL Structure — what works today

| Endpoint | URL Pattern | Purpose |
|---|---|---|
| Portal | design.nyuchi.com/components/\{name\} | Documentation page (live) |
| API | design.nyuchi.com/api/v1/ui/\{name\} | Component metadata + source code (live, shadcn-format) |
| API docs | design.nyuchi.com/api/v1/ui/\{name\}/docs | Use cases, variants, a11y notes (live) |
| API versions | design.nyuchi.com/api/v1/ui/\{name\}/versions | Per-component version history (live) |
| Health | design.nyuchi.com/api/v1/health | Service-wide health check (live, not per-component) |
| Fundi | design.nyuchi.com/api/v1/fundi | Self-healing issue surface (live, not per-component) |

## Planned per-component endpoints

These are documented as canonical in the data-portal contract, but not yet implemented. Tracked in [issue #61](https://github.com/nyuchi/design-portal/issues/61). Until they ship, use the live equivalents above.

| Endpoint | URL Pattern | Status | Today's equivalent |
|---|---|---|---|
| Playground | design.nyuchi.com/playground/\{name\} | planned (#60) | none — components/playground/* parts unwired |
| Source | design.nyuchi.com/source/\{name\} | planned (#61) | `/api/v1/ui/\{name\}` returns `source_code` |
| Changelog | design.nyuchi.com/changelog/\{name\} | planned (#61) | `/api/v1/ui/\{name\}/versions` |
| Health (per-component) | design.nyuchi.com/api/health/\{name\} | planned (#61) | `/api/v1/health` (service-wide) |
| Chaos | design.nyuchi.com/api/chaos/\{name\} | planned (#61) | not yet exposed publicly — L8 chaos lib is in-process |
| Fundi (per-component) | design.nyuchi.com/api/fundi/\{name\} | planned (#61) | `/api/v1/fundi` (cross-cutting issue surface) |

## DOM Integration

Every rendered component includes a `data-portal` attribute pointing at its live portal page:

```html
<div data-slot="nyuchi-section" data-portal="https://design.nyuchi.com/components/nyuchi-section">
```

This allows:
- DevTools inspection to click through to component docs (live today)
- Error tracking to enrich errors with portal links (live today)
- L8 chaos testing to identify components by stable URL (target endpoint #61)
- L9 Fundi to link healing actions to documentation (target endpoint #61)
